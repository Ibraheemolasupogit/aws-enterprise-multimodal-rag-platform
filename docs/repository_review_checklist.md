# Repository Review Checklist

Use this checklist before sharing the repository.

- [ ] README is complete and concise
- [ ] Quickstart commands are documented
- [ ] Example commands are documented
- [ ] `python3 -m pytest` passes
- [ ] Local outputs are generated
- [ ] AWS architecture docs are present
- [ ] Deployment blueprint is present
- [ ] Security model is documented
- [ ] Cost and monitoring model is documented
- [ ] Evaluation strategy and outputs are documented
- [ ] Guardrails are documented
- [ ] Monitoring/reporting artifacts are documented
- [ ] Limitations are clearly stated
- [ ] No credentials or secrets are committed
- [ ] No paid AWS calls are used
- [ ] No deployment claims are made
- [ ] Repository is suitable for portfolio review
